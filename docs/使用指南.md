# pdf_contents_extractor 使用指南

为没有目录标签（书签/outline）的 PDF 添加可点击的层级书签。

## 环境准备

```bash
# 已配置 conda 环境则激活即可
conda activate py311
cd /mnt/d/Python_proj/pdf_contents_extractor
```

依赖：`pymupdf`（PDF 读写）、`thefuzz`（模糊匹配），已安装于 py311 环境。

## 快速开始

```bash
# 第 1 步：提取目录 → JSON
python main.py extract book.pdf -o toc.json

# 第 2 步：编辑 toc.json（修正标题、补全缺失条目、调整页码）

# 第 3 步：将 JSON 目录写回 PDF
python main.py apply book.pdf toc.json -o book_with_toc.pdf
```

## 命令详解

### extract — 从 PDF 提取目录

```bash
python main.py extract <输入PDF> [选项]
```

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `-o`, `--output` | 输出 JSON 文件路径 | `toc.json` |
| `--toc-pages` | 手动指定目录页，如 `3` 或 `3-5` | 自动检测 |
| `--toc-search-pages` | 自动检测时搜索前 N 页 | `20` |
| `--fuzzy-threshold` | 模糊匹配阈值 (0-100) | `85` |

**示例：**

```bash
# 自动检测目录页
python main.py extract book.pdf -o toc.json

# 手动指定目录在第 9 页
python main.py extract book.pdf -o toc.json --toc-pages 9

# 目录跨第 9-10 页
python main.py extract book.pdf -o toc.json --toc-pages 9-10
```

**工作流程：** PDF → 定位目录页 → 解析条目 → 检测印刷页码 → 映射到 PDF 页码 → 导出 JSON。

### apply — 将 JSON 目录写入 PDF

```bash
python main.py apply <输入PDF> <JSON文件> -o <输出PDF>
```

读取编辑后的 JSON，将目录层级写入 PDF outline（书签），生成新的 PDF 文件。**原 PDF 不会被修改。**

### auto — 一键提取并写入

```bash
python main.py auto book.pdf -o book_with_toc.pdf
```

跳过手动编辑环节，直接提取并应用。适合目录清晰、OCR 质量好的 PDF。

## JSON 格式说明

```json
[
  {
    "title": "第一篇 趋势反转趋势成为反趋势",
    "level": 1,
    "page": 89,
    "page_hint": 90,
    "unmatched": false
  },
  {
    "title": "第1章 反转交易范例",
    "level": 1,
    "page": 141,
    "page_hint": 142,
    "unmatched": false
  }
]
```

| 字段 | 说明 |
|------|------|
| `title` | 目录条目标题 |
| `level` | 层级深度：1=章，2=节，3=小节 |
| `page` | **PDF 实际页码（0-based）**，对应阅读器显示页码 = `page + 1` |
| `page_hint` | 换算后的页码（1-based），等同于 PDF 阅读器显示的页数 |
| `unmatched` | `true` 表示未在正文中定位到对应页，需手动补充 |

> **注意：** `page` 和 `page_hint` 已经过自动映射修正——工具会扫描每页底部检测印刷页码，自动将目录中的印刷页码转换为 PDF 实际页码。你编辑 JSON 时主要关注标题修正。

### 页码换算

```
page + 1 = 阅读器中看到的页码 = page_hint
```

例如 `"page": 89` → 在 PDF 阅读器中应跳转到**第 90 页**。

## 手动编辑 JSON 步骤

1. **删除垃圾条目** — OCR 可能将广告行误识别为目录，直接删掉
2. **修正标题文字** — 如 `"第2章 反转动能的标圳 ee"` → `"第2章 反转功能的标的"`
3. **补全缺失条目** — OCR 漏掉的章节手动添加
4. **验证页码** — 在 PDF 阅读器中确认关键章节的页码是否正确
5. **调整层级** — 节标题可设为 `level: 2`，子节 `level: 3`

## 常见问题

**Q: 提示"未找到目录页"？**

用 `--toc-pages` 手动指定。翻看 PDF 找到目录起始页，如目录在第 9 页：

```bash
python main.py extract book.pdf --toc-pages 9
```

**Q: 目录解析结果不完整？**

OCR 处理过的 PDF 文本质量参差不齐，部分条目可能漏掉或合并。需在 JSON 中手动编辑。

**Q: 页码全是错的？**

可能印刷页码映射失败。此时 `page` 值与 PDF 实际页数差一个固定偏移量（如每页都多 9 或都少 9），可以写脚本批量修正。

**Q: 怎么确定正确的页码？**

在 PDF 阅读器中找到该章节，记下显示的页码，填入 JSON 的 `page` 字段（注意：`page` = 显示页码 − 1）。

## 适用场景

- 扫描/OCR 处理后的中文书籍 PDF
- 非影印版、已含文字层的 PDF
- 自带目录页但无书签的出版物 PDF

## 限制

- 纯影印版 PDF（无文字层）无法处理，需先 OCR
- OCR 质量越差，自动提取效果越差，手动编辑工作量越大
- 目录页无"目录/目次/Contents"标签时需要手动指定页码
